# Проект Mesto (Задание 1)

### Локальный запуск
Запуск проект происходит при помощи команды `docker compose up` из корня проекта.
После чего сервис будет доступен по адресу: 👉 http://localhost:3001

### Обоснование используемой технологии
Я выбрал Module Federation вместо Single SPA, 
потому что он позволяет загружать микрофронтенды динамически на уровне Webpack, 
обеспечивая оптимизированный шаринг зависимостей и меньший размер бандлов

### Структура проекта
`auth-microfrontend` отвечает за управление аутентификацией и авторизацией пользователей. 
Он предоставляет механизмы регистрации, входа, выхода и защищенного доступа к ресурсам.

Основные компоненты:
- `Login` – Компонент формы авторизации, позволяющий пользователям вводить логин и пароль.
- `Register` – Компонент формы регистрации нового пользователя.
- `SignOut` – Компонент, осуществляющий очистку пользовательской сессии.
- `ProtectedRoute` – HOC-компонент, защищающий маршруты и перенаправляющий неаутентифицированных пользователей.
- `InfoTooltip` – Всплывающее уведомление с информацией об успешном или неудачном входе/регистрации.
- `AuthRoutes` – Определяет маршруты аутентификации, включая Login, Register и SignOut.
- `AuthContext` – Контекст аутентификации, обеспечивающий глобальное состояние пользователя.
- `useAuth` – Кастомный хук для получения пользователя из контекста.
- `auth` – Файл для работы с серверным API, содержит методы авторизации, регистрации и проверки токена.

`ui-microfrontend` отвечает за унифицированные UI-компоненты, которые могут быть переиспользованы в других микрофронтендах. 
Это позволяет обеспечить единый стиль и поведение интерфейса во всей системе.

Основные компоненты:
- `PopupWithForm` – Универсальный компонент модального окна с формой.

`profile-microfrontend` отвечает за управление профилем пользователя, включая редактирование личных данных и аватара.

Основные компоненты:
- `ProfileSection` – Основной контейнер, отображающий информацию о профиле пользователя (аватар, имя, описание) и кнопки редактирования.
- `EditProfilePopup` – Модальное окно для редактирования имени и описания профиля.
- `EditAvatarPopup` – Модальное окно для обновления аватара пользователя.
- `api`– Файл для работы с серверным API, содержит методы получения и обновления данных профиля.

`card-microfrontend` отвечает за отображение, добавление, удаление и просмотр карточек с изображениями. 
Он позволяет пользователям взаимодействовать с карточками, например, ставить лайки, открывать изображения в увеличенном виде и добавлять новые карточки через всплывающее окно.

Основные компоненты:
- `Card` – Компонент, представляющий отдельную карточку, содержащую изображение, описание и кнопку лайка.
- `CardSection` – Контейнер, отображающий список карточек и управляющий их рендерингом.
- `AddPlacePopup` – Модальное окно для добавления новой карточки, содержит форму с полями ввода.
- `ImagePopup` – Всплывающее окно для просмотра изображения в полном размере.
- `api` – Файл для работы с серверным API, содержит методы получения, создания и удаления карточек.

`host-microfrontend` является главным контейнером, который оркеструет все микрофронтенды.
Основные компоненты:
- `App` – Основной корневой компонент, управляющий глобальным роутингом и подключением микрофронтендов.
- `Header` – Навигационная панель с логотипом, ссылками и кнопкой выхода.
- `Main`– Контейнер, в котором динамически загружаются данные и пользователе и карточках.
- `Footer` – Нижняя часть приложения с копирайтом и ссылками.
